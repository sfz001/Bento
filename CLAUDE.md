## Project Overview

**Bento** — personal macOS menu bar toolkit (Swift + AppKit). A grab-bag of small utilities the owner uses daily. Current modules:
1. **Remote screen-off** — when a RustDesk or macOS Screen Sharing (VNC) connection is detected, the local screen goes black for privacy; on disconnect, the screen restores and locks.
2. **Scroll direction control** — Mos-style independent scroll-direction reversal for mouse vs trackpad.
3. **Window tiling (分屏)** — MaxTo/Moom-style per-monitor split layouts: double-click a titlebar to snap the window into its layout cell (double-click again to restore), or hold ⌘ while dragging a titlebar to snap into the cell highlighted on a grid overlay. Layouts are recursive binary-split trees, edited via a WYSIWYG full-screen overlay editor on each real display, persisted per-display (keyed by CGDisplay UUID) in `~/Library/Application Support/Bento/config.json`.
4. **Menu bar icon manager (菜单栏图标管理)** — Bartender/Ice-style selective hide/show of menu bar icons: menu opens a checklist window (NSTableView) listing every third-party icon by owning app **plus movable system modules** (Battery/Wi-Fi/User Switching; Clock and Control Center are pinned by the system and excluded) **plus Bento's own icon** (sortable, not hideable); unchecking hides an icon into the system "«" overflow section, checking shows it again; rows are **drag-reorderable** and the row order is applied to the real menu bar (persisted as `MenuBarIconOrder`). All moves are done by rewriting `com.apple.MenuBarAgent`'s `TrailingItemPreferredPositions` plist (macOS 27 mechanism — no synthetic drags); hiding assigns positions left of everything visible so the crowded bar (notch + wide ticker icons) overflows them — there is no forced-hide primitive on 27, so on a roomy bar hidden items can become visible again. Hidden keys persist in UserDefaults (`HiddenMenuBarItemKeys`) and change **only** through the manager UI.

The project began as `RustDeskScreenOff` (remote screen-off only) before growing into Bento; repo/dir/app/bundle are all `Bento` now.

## Build & Run

```bash
./build_app.sh             # Compile arm64, package .app bundle (icon from Assets/AppIcon.icns)
./build_app.sh --check     # Seconds-fast typecheck only — no binary, no bundle
BUILD_UNIVERSAL=1 ./build_app.sh   # arm64 + x86_64 universal binary
open Bento.app             # Run
```

Build requires macOS 14.0+ SDK. Uses `swiftc` directly (no Xcode project/SPM). Frameworks: AppKit, CoreGraphics, IOKit (IOKit is used only by ScrollReverser).

## Architecture

Code lives in `Sources/`, one file per module, compiled together by `build_app.sh` (`swiftc Sources/*.swift`; top-level entry statements must stay in `main.swift`):

- `Sources/Support.swift` — shared `runProcess` helper + `ErrorLog`
- `Sources/ScreenController.swift`, `Sources/ScrollReverser.swift`, `Sources/WindowTiling.swift`, `Sources/MenuBarIconManager.swift`, `Sources/AppDelegate.swift` — the modules below
- `Sources/main.swift` — single-instance guard, uncaught-exception logging, NSApplication bootstrap

File-scope `private`/`fileprivate` declarations don't cross these file boundaries — anything shared between modules belongs in `Support.swift` as internal.

- **`ScreenController`** — Screen blackout via `CGSetDisplayTransferByFormula` (gamma to zero), restore via `CGDisplayRestoreColorSyncSettings`, lock via `open -a ScreenSaverEngine`. Multi-monitor mirroring via `CGConfigureDisplayMirrorOfDisplay` (merge on connect, restore on disconnect). Before mirroring, it persists a display mirror snapshot in `UserDefaults` (`BentoMirrorSnapshot`) so disconnect/relaunch can restore the pre-connection layout instead of relying only on process memory; the menu also exposes 恢复扩展显示器 as a manual fallback. Resolution switching to 1512x982 HiDPI on connect (saves/restores original mode). Dock repositioning (moves to left on connect, restores original position/autohide on disconnect). Gamma is invisible to ScreenCaptureKit so remote viewers see normal desktop.

- **`ScrollReverser`** — Independent scroll-direction control (Mos-style). Installs one session event tap for both two-finger gesture detection and scroll-wheel delta rewriting. Trackpad vs mouse follows Scroll Reverser’s approach: non-continuous scrolls are mouse, recent 2+ finger gesture scrolls are trackpad, momentum continues with the previous source. Reverses line/fixed-point/point delta axes when enabled, setting point deltas last to preserve smooth scrolling, and also rewrites the underlying IOHID scroll fields for apps that read those. Defaults: mouse reversed (ON), trackpad untouched (OFF). Persists toggles in `UserDefaults` (`ReverseMouseScroll`, `ReverseTrackpadScroll`). Self-heals on `.tapDisabledByTimeout`, rebuilds the tap on wake/session-active, and a 5s watchdog re-enables it if found disabled. Requires Accessibility and Input Monitoring permissions; if denied, Bento shows a warning alert, exposes menu actions to open the relevant System Settings panes, disables scroll toggles with "(Grant Permissions)", and allows retrying without relaunch.

- **`AppDelegate`** — Orchestrates everything. Every 3 seconds polls for active connections: `pgrep -fi "rustdesk.*--cm"` for RustDesk, and `netstat -an -p tcp` parsed in Swift (local column ending `.5900` + ESTABLISHED) for macOS Screen Sharing — no shell/awk pipeline. Idle mirroring cleanup is edge-triggered (first poll + connected→disconnected), not every tick. On connect: enable mirroring → switch resolution → dock to left → black screen. On disconnect: restore screen → restore resolution → restore dock → disable mirroring → lock. Manages the all-Chinese NSStatusItem menu (FileVault 免密重启, screen-off controls, etc.) — grouped with macOS 14 `sectionHeader` items (no separators before headers — the header IS the section divider; only the trailing general section keeps one); note NSMenuItem.image is NOT rendered in status-item menus on macOS 27 (verified with a minimal reproducer), the symbol assignments are kept for future OS versions, scroll-reverse toggles, and the tiling section. Login auto-start uses `SMAppService.mainApp` (registered on first launch; afterwards the 开机自启 menu toggle owns it); a one-time migration unloads and deletes the old hand-written `~/Library/LaunchAgents/com.sz.bento.plist`.

- **Tiling module** (`MARK: - Window Tiling` section) — key pieces:
  - `CoordConv` — the ONLY place AppKit (bottom-left origin) ↔ CG/AX (main-screen top-left origin) conversion happens. Never flip y elsewhere.
  - `LayoutNode` — recursive binary-split tree (`cell` / `split(dir:V|H, ratio, a, b)`), defensive Codable (bad ratio→0.5, missing node→cell, corrupt JSON→default + error.log).
  - `TilingConfig` — JSON persistence; display layouts keyed by CGDisplay UUID (`DisplayKeys`), never displayID/screen index.
  - `WindowManager` — CGWindowList geometry prefilter (titlebar band, 28pt; window list cached ~80ms so ⌘-click bursts and the ⌘-double-click double-query share one enumeration) before any AX call; AX frame set with read-back + one retry; snappable = standard subrole, not fullscreen/minimized, size-settable.
  - `TilingController` — session event tap (leftMouseDown/Up/Dragged; mouseMoved is deliberately NOT subscribed — ⌘-drag emits leftMouseDragged, and a mouseMoved subscription would wake the process on every mouse move for nothing) that *swallows* the 2nd click of titlebar double-clicks (plus its paired mouse-up, plus same-spot 3rd+ clicks inside the double-click window); the window's left ~80pt traffic-light zone is EXCLUDED from snap/swallow — eating the 2nd click there breaks the close/minimize buttons' multi-click tracking (close button appears dead) while snapping the window away; modifier+titlebar-drag shows grid overlays (`ignoresMouseEvents`, `.screenSaver` level) and snaps on release. Snap memory stores (original, actual-snapped) frames per CGWindowID (`_AXUIElementGetWindow` private API); restore clamps to nearest screen if the original display is gone. Watchdog re-enables/rebuilds the tap; wake/unlock rebuilds it and resets all click state; `didChangeScreenParameters` closes the editor and invalidates display-key cache.
  - `LayoutEditorSession` — one borderless editor window per screen covering exactly `visibleFrame` (menu bar/Dock stay clickable), floating toolbar with grouped buttons (保存并应用 = accent primary, rightmost; no `.hudWindow` — HUD restyles buttons and swallows bezelColor), right-click split/merge menu, divider dragging clamped to 0.08–0.92 with hover highlight + resize cursor rects, hint pill drawn at the bottom (top belongs to the toolbar), Esc cancels / Return saves (local keyDown monitor — the panel's keyEquivalent can't fire while an editor window is key); only 保存并应用 writes config.
  - Uncaught exceptions append to `~/Library/Application Support/Bento/error.log` (serialized writes; rolls to `error.log.1` past 256KB; the crash handler uses `ErrorLog.logSync` — an async enqueue would be lost when the process aborts); single-instance enforced in main.

- **Menu bar icon manager** (`MARK: - Menu Bar Icon Manager` section) — targets the macOS 27 menu bar architecture and no-ops when the MenuBarAgent process is absent, i.e. on macOS ≤26 (the 26-era ControlCenter-hosted / synthetic-⌘-drag "scromble" implementation was removed; see git history if ever needed). Key facts, all verified on 27.0:
  - macOS 27 hosts the whole menu bar in a dedicated `MenuBarAgent` process: no more per-icon layer-25 CG windows, so CG-window enumeration finds nothing. Item layout lives in `com.apple.MenuBarAgent`'s `TrailingItemPreferredPositions` dict: key `status:<signing-id>::<autosaveName>` (bundleID for properly signed apps, executable name for ad-hoc — Bento writes both candidate keys for its own item, `BentoMain`) or `module:<Name>` for system modules; value = preferred distance from the right edge (bigger = further left). External writes apply live within ~0.5s (cfprefsd) — no agent restart, no synthetic events.
  - System modules are managed through the same `module:` entries (AX id ↔ dict key via `moduleKeyMap`; instance suffixes like `module:BentoBox-0` handled). Verified movable/hideable: Battery, WiFi, UserSwitcher. Verified PINNED (writes silently ignored, frame never moves): Clock and Control Center (`module:BentoBox` — Apple's internal name for it, coincidentally) — these are in `pinnedModules` and never shown as rows. All managed items (third-party + modules + `bento:main`) share one unified position grid (base 100, step 8); order adoption runs every pass — the actual bar layout is the source of truth for ordering, never the persisted list (enforcing a stale persisted order silently rearranges the user's bar).
  - System layout rule is **greedy hole-skipping**: items are packed right-to-left in ascending position order; an item that doesn't fit is skipped into the "«" overflow chevron and packing continues with later items. There is NO truncation — an oversized item only hides itself, it blocks nobody (verified: a 10000pt item next to a 30pt probe left every later item visible). The old 10000pt-hider design is therefore useless on 27 and was removed; hiding = assigning hidden items positions (1100+8·j) left of all visible ones (base 100+8·i) and relying on genuine crowding to overflow them. New items with no dict entry start in overflow on a crowded bar.
  - **The agent spontaneously rewrites the whole dict with self-computed actual distances after status-item register/deregister churn** (e.g. any Bento relaunch), including transient overflow/expanded-strip coordinates that can exceed any threshold. Position values must therefore NEVER be adopted as hide/show intent — doing so mass-corrupted `HiddenMenuBarItemKeys` once (adopting expanded-strip coordinates as "user dragged these into hiding"). Hide/show intent comes exclusively from the manager UI; the dict is only trusted for the left-to-right ORDER of visible items (its actuals faithfully reflect the real bar). Semantic drift (should-be-hidden sitting visible, wrong order, missing entries) is corrected by one rewrite on the next pass, and every rewrite logs its reasons to error.log.
  - Entry-key resolution writes to exactly ONE prefix per app (the one with existing dict entries; bundleID synthesized only when none exist — nearly all third-party menu apps are properly signed; Bento's own ad-hoc prefix is the executable name). Pairing ladder: unique existing key → index pairing on equal counts → Item-N suffix matching → skip writes + log once per app (writing to a wrong key is worse than not writing). Phantom keys under the losing prefix are garbage-collected on the next rewrite. The overflow chevron exposes NO AXPress action (verified on both states and the full ancestor chain, error -25206), so collapsing the expanded strip stays a coordinate click, guarded by frame sanity checks against AX ghost frames.
  - Menu-quit (退出 Bento) calls `prepareForQuit()`: hidden items' entries are written back to a visible band so icons don't stay sunk in overflow after an uninstall; `HiddenMenuBarItemKeys` is kept, so a relaunch re-hides them. Deliberately NOT in `applicationWillTerminate` — restarts/shutdowns pass through there and each would add status-item churn (the known agent-rewrite trigger).
  - Identity still comes from each app's `AXExtrasMenuBar` (title/desc); persistence keys are `bundleID|index`, never titles (WeChat badge titles are volatile). Dict entries are matched per app by prefix; when entry count equals item count they pair in sorted order (covers custom autosave names like `systemuiserver::Siri`, `campo::Item-1`).
  - Enforcer runs every 3s on a serial queue, gated by a (positions-dict + running-apps) signature, with a 60s forced full pass as fallback. Per-pass AX cost is bounded: pids confirmed to have no menu-bar extras are skipped on regular passes (re-probed only on the 60s pass), CFBundleName lookups are cached, and `iconOrder`/`iconNames` are batch-assigned so UserDefaults is written at most once per pass (`iconNames` stores only the stable app name, never volatile ticker/badge text). Visible-item ORDER is adopted from the dict every pass (manual ⌘-drags and agent rewrites both reflect the real bar; never enforce a stale persisted order — that silently rearranges the user's bar). Bento only rewrites the dict on semantic mismatch (should-be-hidden visible, wrong visible order, missing entries), never over numeric drift — this avoids write-fights with the agent. UI-triggered rounds set `suppressAdoptionOnce` so a just-dragged row order isn't immediately overwritten by pre-drag positions.

- **Main entry** — Creates `NSApplication`, sets delegate, calls `app.run()` (no storyboard/NIB).

## Key Technical Details

- `LSUIElement=true` in Info.plist hides Dock icon (menu bar only)
- Gamma stays black as long as the process runs; killing the app restores normal display
- Bundle ID: `com.sz.bento`
- LaunchAgent plist written to `~/Library/LaunchAgents/com.sz.bento.plist`
